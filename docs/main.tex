\begin{abstract}
%═══════════════════════════════════════════════════════════════
Any energy-based model whose energy decomposes over local factors has a Gauss--Newton Hessian that decomposes additively over those same factors.
This Hessian---taken jointly over latent states and parameters---is the complete local description of the energy landscape: it governs how fast inference converges, how quickly parameters are identified, and how states and parameters couple.
Each factor contributes a PSD block whose directional structure is determined by the transform type, not by the random weight values.
We use this object for structure learning.
The score is the alignment between the Hessian's directional structure and the task: $\mathrm{tr}(\bm{H}_S^{-1}\bm{\Sigma}_{\mathrm{task}})$, an A-optimal criterion that is low when the graph provides precision in the directions the task demands.
A spectral gate $\varepsilon = 1/\eta T$ ties the score to the computational budget, making it discriminate between graphs with identical connectivity but different spectral quality.
The algorithm performs backward elimination on a supergraph, requires only the Jacobians from a single forward pass, and for linear transforms is exact.
\end{abstract}

%═══════════════════════════════════════════════════════════════
\section{The Energy Hessian}
%═══════════════════════════════════════════════════════════════

An energy-based model on a factor graph $\cG$ assigns energy $E(\bx;\btheta,\by) = \sum_e E_e$ to configurations of free states $\bx \in \R^D$, with parameters $\btheta \in \R^P$ and clamped data $\by$.
Each factor $e$ has a prediction residual $\bm{r}_e = f_e(\bm{v}_{\mathrm{src}};\btheta_e) - \bm{v}_{\mathrm{tgt}}$ and a local energy $E_e = \frac{1}{2}\bm{r}_e^\top\bm{\Lambda}_e\bm{r}_e$.

The Gauss--Newton approximation to the Hessian of $E$ with respect to all unknowns $\bz = (\bx, \btheta)$ decomposes over factors:
\begin{equation}\label{eq:joint}
    \bm{H}_\cG = \sum_e \bm{J}_e^\top \bm{\Lambda}_e\,\bm{J}_e, \qquad \bm{J}_e = \frac{\partial \bm{r}_e}{\partial \bz} \in \R^{d_e \times (D+P)}.
\end{equation}
For linear transforms, this is exact.
Each factor contributes a PSD term.
The sum is the complete local description of the energy landscape.

This single object governs everything.
The $\bx$-$\bx$ block $\bm{H}_{xx}$ is the inference precision: it determines how fast gradient descent on $\bx$ converges.
The Schur complement $\bm{H}_{\theta\theta} - \bm{H}_{\theta x}\bm{H}_{xx}^{-1}\bm{H}_{x\theta}$ is the Fisher information after marginalising states: it determines how quickly parameters are identified once inference has settled.
The cross block $\bm{H}_{x\theta}$ couples inference and learning: it determines whether resolving states also constrains parameters.

An edge that contributes only to $\bm{H}_{xx}$ provides inference capacity but no learning signal.
An edge that contributes to $\bm{H}_{x\theta}$ links inference to learning: adjusting states also identifies parameters.
An edge that contributes only to $\bm{H}_{\theta\theta}$ provides parameter curvature but no coupling to states.
The decomposition over factors makes each edge's role visible before any inference or learning has occurred.

\paragraph{Structure learning from the Hessian.}
The question ``which edges does this graph need?'' becomes: which subset $S$ of factors produces a Hessian $\bm{H}_S$ with good geometry for the task?
The task enters through the clamping pattern (which variables carry data) and the data distribution (which prediction errors are large).
The transform types enter through the Jacobians (which directions each factor provides precision in).
Structure learning is the problem of matching one to the other.

\section{Scoring a Graph}
%═══════════════════════════════════════════════════════════════

\subsection{The score}

For an edge set $S$, let $\bm{H}_S = \sum_{e \in S}\bm{J}_e^\top\bm{\Lambda}_e\bm{J}_e + \varepsilon\bm{I}$ be the regularised Hessian.
We score $S$ by the A-optimal criterion:
\begin{equation}\label{eq:score}
    J(S) = \mathrm{tr}\!\left(\bm{H}_S^{-1}\,\bm{\Sigma}_{\mathrm{task}}\right).
\end{equation}
Here $\bm{\Sigma}_{\mathrm{task}} \in \R^{(D+P)\times(D+P)}$ encodes which directions in the joint $(\bx,\btheta)$ space the task demands precision in.
It is defined empirically: run a forward pass on the supergraph with a probe batch, collect the residuals $\bm{r}_{e,n}$ at data-facing edges, and set
\begin{equation}\label{eq:sigma}
    \bm{\Sigma}_{\mathrm{task}} = \frac{1}{N}\sum_{n=1}^N \bm{J}_{\mathrm{data}}^\top\bm{r}_{\mathrm{data},n}\,\bm{r}_{\mathrm{data},n}^\top\bm{J}_{\mathrm{data}},
\end{equation}
where $\bm{J}_{\mathrm{data}}$ stacks the Jacobians of data-facing edges and $\bm{r}_{\mathrm{data},n}$ stacks their residuals.
This projects data variance into the joint $(\bx,\btheta)$ space: directions that are irrelevant to the data get zero task variance.

The score $J(S)$ is the expected residual variance under the task distribution, given the precision provided by the graph.
It is low when the Hessian's eigenvectors align with the task's principal directions---when the graph provides precision where the task demands it.

\subsection{What the score captures}

Let $\bm{H}_S = \bm{U\Lambda U}^\top$.
Then:
\begin{equation}\label{eq:decomp}
    J(S) = \sum_i \frac{\sigma_i}{\lambda_i + \varepsilon}, \qquad \sigma_i = \bm{u}_i^\top\bm{\Sigma}_{\mathrm{task}}\,\bm{u}_i.
\end{equation}
This separates into two failure modes:
\begin{itemize}
    \item \textbf{Coverage gap}: directions with $\lambda_i \approx 0$ and $\sigma_i > 0$.
    The task demands precision here but the graph provides none.
    These are structural deficits---missing edges or wrong connectivity.
    \item \textbf{Conditioning penalty}: directions with $0 < \lambda_i \ll \varepsilon$ and $\sigma_i > 0$.
    The graph provides some precision but too little relative to the budget.
    These are ill-conditioned paths---deep chains with exponentially spread spectra.
\end{itemize}
Directions with $\sigma_i = 0$---those irrelevant to the task---contribute nothing regardless of $\lambda_i$.
Edges that provide precision only on task-irrelevant directions have zero marginal contribution and are pruned for free.

\subsection{Why it works over both states and parameters}

Because $\bm{H}_S$ is the joint Hessian and $\bm{\Sigma}_{\mathrm{task}}$ is projected into the joint space, the score simultaneously rewards:
\begin{itemize}
    \item Good inference geometry: the $\bx$-$\bx$ block is well-conditioned in directions that matter for resolving data-facing predictions.
    \item Good learning geometry: the parameter-side Schur complement is well-conditioned, meaning data constrains parameters through the state-parameter coupling.
    \item Useful coupling: the cross block $\bm{H}_{x\theta}$ connects inference to learning, so resolving states also identifies parameters.
\end{itemize}
An edge that helps only inference (adds to $\bm{H}_{xx}$ in task-relevant directions) improves the score.
An edge that helps only learning (adds to $\bm{H}_{\theta\theta}$) also improves the score, if the task covariance has nonzero parameter-side variance.
An edge that helps both is rewarded most.
The A-optimal criterion automatically balances these contributions through the trace.

\subsection{The spectral gate}

Setting $\varepsilon = 1/\eta T$ ties the regularisation to the computational budget---$T$ steps of gradient descent with step size $\eta$.
At convergence ($\varepsilon \to 0$), the score depends only on whether each task-relevant direction has \emph{any} precision.
All connected graphs score identically.

At finite $T$, the gate penalises directions where $\lambda_i < 1/\eta T$: the graph provides precision but inference/learning cannot exploit it within budget.
A deep chain has exponentially spread eigenvalues; most fall below the gate.
A skip connection has $O(1)$ eigenvalues; they sit above the gate.
The computational budget is the mechanism that makes structure learning non-trivial.

%═══════════════════════════════════════════════════════════════
\section{What Each Edge Contributes}
%═══════════════════════════════════════════════════════════════

Each edge's Hessian contribution $\bm{J}_e^\top\bm{\Lambda}_e\bm{J}_e$ has a directional structure determined by the transform type.
This structure is algebraic---a property of the transform \emph{class}, not of the specific random weight draw.
The score selects edges whose combined directional structure best matches the task.

\paragraph{Dense layers.}
$f(\bm{v}) = \bm{W}\bm{v}$: the Jacobian $\bm{J}_e$ depends linearly on $\bm{W}$, giving $\bm{J}_e^\top\bm{J}_e$ with isotropic eigenvectors (Haar-distributed).
No task direction is preferred.
Dense layers provide capacity without directional bias---a neutral baseline.

\paragraph{Convolutions.}
$f(\bm{v}) = \bm{W}*\bm{v}$: the Jacobian is block-circulant, diagonalised by the DFT regardless of filter values.
The Hessian contribution has eigenvectors locked to the Fourier basis; only eigenvalues are random.
On natural images, where $\bm{\Sigma}_{\mathrm{task}}$ concentrates variance at low frequencies, this Fourier structure aligns with the task and produces a lower $J(S)$ than isotropic alternatives.

\paragraph{Attention.}
At random init with large key dimension, softmax approaches uniform weights and the Jacobian approximates mean-pooling.
The Hessian contribution emphasises position-averaged directions.
On data with positional structure, this alignment is favourable.

\paragraph{Depth.}
A chain $\bm{v}_0 \to h_1 \to \cdots \to h_L \to \bm{v}_L$: the composed Jacobian has singular values spreading as $e^{cL}$.
The joint Hessian inherits this spread---most eigenvalues fall below the spectral gate.
This is the Hessian-level description of vanishing gradients, visible before any training.

\paragraph{Skip connections.}
A skip edge adds a PSD term with $O(1)$ eigenvalues---no multiplicative composition.
It raises the smallest eigenvalues of $\bm{H}_S$, directly correcting the spectral damage from depth.

\paragraph{Coupling edges.}
An edge whose residual depends on both $\bx$ and $\btheta$ (which is typical: $r_e = f_e(v_{\mathrm{src}};\theta_e) - v_{\mathrm{tgt}}$ involves both) contributes to the cross block $\bm{H}_{x\theta}$.
The magnitude of this cross contribution determines how much inference and learning are coupled---how much resolving the state at one end of the edge also constrains the parameters of the transform.
Edges with large cross coupling are structurally valuable because they link the two optimisation problems.

\section{Algorithm}
%═══════════════════════════════════════════════════════════════

Given per-edge Jacobians $\bm{J}_e$ (one forward pass) and the task covariance $\bm{\Sigma}_{\mathrm{task}}$ (one probe batch), the score of any edge set is computable in $O((D{+}P)^3)$.

\begin{algorithm}[t]
\caption{Backward Elimination via Hessian Matching}
\label{alg:main}
\begin{algorithmic}[1]
\REQUIRE Supergraph with $M$ edges, probe batch, gate $\varepsilon = 1/\eta T$
\STATE Compute $\bm{J}_e$, $\bm{\Lambda}_e$ for all edges (one forward pass)
\STATE Compute $\bm{\Sigma}_{\mathrm{task}}$ from data-facing residuals~\eqref{eq:sigma}
\STATE $S \leftarrow \{1,\dots,M\}$; \; $J_{\mathrm{curr}} \leftarrow J(S)$
\REPEAT
    \FOR{each non-data-facing edge $e \in S$}
        \STATE $J_e \leftarrow J(S \setminus \{e\})$ \hfill\COMMENT{$O((D{+}P)^3)$}
    \ENDFOR
    \STATE $e^* \leftarrow \arg\min_e J_e$
    \IF{$J_{e^*} > J_{\mathrm{curr}} + \delta$}
        \STATE \textbf{break}
    \ENDIF
    \STATE $S \leftarrow S \setminus \{e^*\}$; \; $J_{\mathrm{curr}} \leftarrow J_{e^*}$
\UNTIL{no removable edges remain}
\RETURN $S$, decomposition~\eqref{eq:decomp}
\end{algorithmic}
\end{algorithm}

Data-facing edges define $\bm{\Sigma}_{\mathrm{task}}$; removing them changes the problem.
All other edges are candidates for removal.
Edges providing precision only on directions where $\sigma_i = 0$ contribute nothing and are removed first.
The algorithm stops when every remaining removal would increase the score beyond tolerance.

The returned decomposition~\eqref{eq:decomp} is a diagnostic certificate: coverage gaps identify which directions lack precision (missing paths), and conditioning penalties identify where precision exists but is too weak (paths that are too deep or poorly conditioned).

\paragraph{Scaling.}
For small systems ($D + P \lesssim 500$), the full Hessian is practical.
For larger systems, two reductions are available.
First, restrict to the state-only Hessian $\bm{H}_{xx}$ (dropping parameter dimensions), recovering the pure-inference view at $O(D^3)$.
Second, maintain $\bm{H}_S^{-1}$ and update via Woodbury when removing an edge, giving per-candidate cost $O((D{+}P)^2 d_e)$.
Both preserve the additive decomposition and the A-optimal score.

\paragraph{Views by elimination.}
The joint Hessian contains multiple structure-learning criteria, accessible by eliminating different variable blocks:
\begin{itemize}
    \item \textbf{Inference view}: score $\bm{H}_{xx}$ alone---does the graph resolve states well?
    \item \textbf{Learning view}: eliminate $\bx$, score the Schur complement $\bm{H}_{\theta\theta} - \bm{H}_{\theta x}\bm{H}_{xx}^{-1}\bm{H}_{x\theta}$---does the graph make parameters identifiable?
    \item \textbf{Joint view}: score $\bm{H}_S$ directly---does the graph have good overall geometry?
\end{itemize}
All three use the same additive decomposition, the same algorithm, and the same A-optimal criterion.
They differ in which submatrix is inverted.
The joint view is the most complete; the inference and learning views are projections of it.

\section{Related Work}
%═══════════════════════════════════════════════════════════════

The Gauss--Newton Hessian and its additive decomposition over factors is the basis of Gaussian graphical models~\citep{lauritzen1996graphical,koller2009pgm}, expectation propagation~\citep{minka2001ep}, and bundle adjustment in computer vision~\citep{triggs2000bundle}.
In all cases, the sparse block structure is exploited for efficient inference.
We use the same structure for a different purpose: scoring and selecting the graph itself.

Bayesian model reduction~\citep{friston2018bmr,friston2016group} scores reduced models by variational free energy, penalising large eigenvalues (over-commitment).
Our score penalises small eigenvalues relative to the spectral gate (under-conditioning).
Both share the invert-once-reduce-many structure.

Training-free NAS methods~\citep{chen2021tenas,shu2022nasi} use NTK-derived quantities~\citep{jacot2018neural}, but the NTK conflates component spectra through end-to-end composition~\citep{mok2022demystifying,zhou2024vintk}.
The edge-decomposed Hessian avoids this: each transform's directional contribution is visible locally.

The A-optimal criterion is standard in experimental design~\citep{pukelsheim2006optimal} and sensor placement~\citep{krause2008near}, where greedy elimination has approximate submodularity guarantees.

\section{Discussion}
%═══════════════════════════════════════════════════════════════

The energy Hessian of a factor graph, taken jointly over states and parameters, is the complete local description of the model's geometry.
It decomposes additively over edges.
Each edge's contribution carries a directional signature determined by its transform type---Fourier for convolutions, isotropic for dense, position-averaged for attention.
Structure learning reduces to selecting the edge set whose combined signature best matches the task.

The spectral gate $\varepsilon = 1/\eta T$ makes this non-trivial: at infinite budget all connected graphs look the same; at finite budget the conditioning of the Hessian discriminates between them.
The A-optimal score balances coverage (does the graph provide precision in all task-relevant directions?) against conditioning (is that precision large enough to be exploited within budget?)---and does so simultaneously over states and parameters, capturing both inference quality and parameter identifiability in a single number.

The method is simple: one forward pass for Jacobians, one probe batch for the task covariance, backward elimination with $O((D{+}P)^3)$ per step.
It requires no training, no inference loop, and no label information beyond the probe batch.

The central empirical question is whether the score---which measures structural capacity at a single random linearisation point---predicts post-training performance.
The framework predicts it should: convolutions should beat dense on images (Fourier alignment), skip connections should beat depth at finite $T$ (spectral conditioning), and dead-end edges should contribute nothing (zero task-relevant precision).
Each prediction follows from a specific algebraic property of the transforms interacting with the task covariance.
